# Planos Técnicos (`plans/`)

Esta pasta guarda o **plano técnico** derivado da especificação.

Você vai criar aqui, no Module 03:

- `weather-app-plan.md` — arquitetura, stack, modelo de dados, fluxo de dados,
  decisões técnicas e estratégia de testes.

O plano **consome** a spec (`specs/weather-app-spec.md`) e **alimenta** a
quebra de tarefas (`tasks/`).
