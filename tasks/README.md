# Backlog de Tarefas (`tasks/`)

Esta pasta guarda o **backlog granular** derivado do plano técnico.

Você vai criar aqui, no Module 04:

- `weather-app-tasks.md` — lista de tarefas pequenas, independentes e
  testáveis, organizadas por entrega e prontas para serem delegadas a um agente.

As tarefas **consomem** o plano (`plans/weather-app-plan.md`) e guiam a
implementação (Modules 05–08).
